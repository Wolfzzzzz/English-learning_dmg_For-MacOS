## 0. 背景与关键假设（已知上下文）

- **数据来源**：客户桌面 PDF《雅思词汇绿宝书(1).pdf》（路径 `/Users/zzn/Desktop/雅思词汇绿宝书(1).pdf`，共 518 页）。该书按 list 组织词汇，每词含英文与中文释义。工程师阶段需从 PDF 抽取词库并生成结构化数据文件（JSON/plist）随应用打包。
- **技术栈**：Swift + SwiftUI，目标 macOS 13+。核心调度逻辑（课程推进、单词筛选、批改）须独立成可单测模块，UI 层调用之。
- **主理人默认假设（已采纳，待确认项见第 5 节）**：
  1. "初中课标范围"采用一份内置标准初中英语课标词表（约 1600 词）做排除，做成可替换的数据文件。
  2. 应用本地持久化（UserDefaults / 文件）记录每日已完成练习与历史报告；课程推进仅由日期计算得出（无需联网/账号）。

> ⚠️ **数据就绪风险（新增）**：经技术核验，源 PDF 为**纯图像扫描件**（518 页、零可提取文本、每页均为内嵌图片）。词库抽取**必须依赖 OCR**，其准确率与工作量需提前评估。详见第 5 节 Q7。

## 1. 产品目标

为以中国大陆初中（及更早）英语基础为起点、正在自学备考雅思的用户，提供一款 macOS 原生、零账号零联网的每日单词默写（中译英主动回忆）练习工具。通过将《雅思词汇绿宝书》按 list 组织为"每日一课（2 个 list）"的轻量节奏，自动剔除初中课标范围内的简单词，帮助用户在碎片时间高效巩固高阶雅思词汇，并以即时批改与历史报告量化记忆效果、驱动持续复习。

---

## 2. 用户故事

1. **作为**雅思自学者，**我希望**应用每天自动推送一课（从当日 2 个 list 中筛选出的 50 个非初中词），**以便**无需自行规划即可坚持每日练习。
2. **作为**备考用户，**我希望**每题看到中文释义并在下划线上填写英文，**以便**训练中译英的主动回忆能力而非被动识别。
3. **作为**粗心用户，**我希望**写完点「校对」即可看到正确率、错词及正确答案，**以便**立刻定位薄弱点并订正。
4. **作为**长期用户，**我希望**能回看历史练习报告与错词本、查看进度，**以便**做针对性复习与自我激励。
5. **作为**听觉型学习者（P2），**我希望**能点击单词听发音，**以便**同时强化读音与拼写的关联。

## 3. 需求池

优先级定义：**P0 必须有** / **P1 应该有** / **P2 可有**。

### P0 — 核心闭环（必须有）

| 编号 | 需求 | 验收标准 |
|------|------|----------|
| P0-1 | **课程模型与 UTC+8 每日自动推进** | 以 list 为单位，每 2 个 list 为一课；起点为 list17（含）；按 UTC+8 时区、以"天"为最小粒度推进。给定任意 UTC+8 日期，能确定性计算出当日对应 2 个 list（公式：第 d 课 = list(17+2d) 与 list(17+2d+1)，d 为自锚定日起的天数偏移），跨日/跨周连续无跳变、无重复、无遗漏。 |
| P0-2 | **每日非初中课标词筛选** | 从当日 2 个 list 的词库中，基于内置初中课标词表（~1600 词，可替换数据文件）排除后，选取 50 个单词供练习。选出的词**均不在**初中词表内，且**仅来源于**当日 2 个 list。若当日可用非初中词不足 50，则取全部并在 UI 提示实际数量。 |
| P0-3 | **默写界面（中文释义 + 下划线输入框）** | 每题展示单词的中文释义（含词性标注），下方为下划线样式输入框，输入字数不限；50 题可滚动浏览，焦点/回车可在题间顺畅切换。 |
| P0-4 | **「校对」批改与报告** | 界面底部设「校对」按钮；点击后逐题比对用户答案与标准答案，生成报告，至少包含：**正确率**（正确数/总数，百分比）、**错误单词列表**（中文释义 / 用户答案 / 正确答案）。报告在底部区域展开展示。 |

### P1 — 留存与复习（应该有）

| 编号 | 需求 | 验收标准 |
|------|------|----------|
| P1-1 | **本地持久化历史报告** | 每日完成练习后，报告（日期、课次、正确率、错词明细）持久化于本地（UserDefaults/文件），可回看往日报告列表与详情。 |
| P1-2 | **错词本 / 复习模式** | 自动归集历次错词，提供"仅复习错词"的练习入口，可重做并再次批改。 |
| P1-3 | **进度可视化** | 展示已学 list 范围、连续练习天数、累计正确率、总练习词数等概览。 |

### P2 — 增强体验（可有）

| 编号 | 需求 | 验收标准 |
|------|------|----------|
| P2-1 | **发音朗读** | 单词可点击播放发音（TTS 或音频资源），支持逐题朗读。 |
| P2-2 | **题目随机打乱** | 每次练习可对 50 题顺序随机重排。 |
| P2-3 | **错词加权复现** | 历史错词在后续练习中以更高概率被抽中。 |
| P2-4 | **报告导出** | 支持将单日/累计报告导出为 PDF / CSV / 纯文本。 |

## 4. UI 设计稿（主界面布局）

采用单窗口、自上而下三段式布局；窗口适配 macOS 13+ 标准尺寸，支持滚动。

```
┌──────────────────────────────────────────────────────────────┐
│  ① 顶部：课程与日期信息区（固定）                               │
│     第 N 课（List 17–18）   日期：2026-07-14 (UTC+8)            │
│     今日单词：50 个   状态：未开始 / 已完成 ✓                    │
├──────────────────────────────────────────────────────────────┤
│  ② 中部：题目区（可滚动）                                       │
│  ┌────────────────────────────────────────────────────────┐  │
│  │ 1.  adj. 重要的，重大的                                  │  │
│  │     _______________   ← 下划线输入框（字数不限）          │  │
│  │ 2.  n. 环境，周围状况                                    │  │
│  │     _______________                                     │  │
│  │ 3.  v. 建立，设立                                        │  │
│  │     _______________                                     │  │
│  │            … 共 50 题 …                                  │  │
│  └────────────────────────────────────────────────────────┘  │
├──────────────────────────────────────────────────────────────┤
│  ③ 底部：操作与报告区（固定）                                   │
│  ┌──────────────────────────────────────────────────────┐    │
│  │              [      校  对      ]   ← 宽按钮            │    │
│  └──────────────────────────────────────────────────────┘    │
│  ── 批改报告（点击「校对」后展开）──                            │
│  正确率：82%（41 / 50）                                        │
│  错误单词（9）：                                                │
│    2. 环境，周围状况  → 你填：enviroment   正确：environment   │
│    7. 建立，设立      → 你填：（空）        正确：establish    │
│    …                                                          │
└──────────────────────────────────────────────────────────────┘
```

**布局要点**
- ①区展示当前课次、对应 list 区间、UTC+8 日期、当日单词数与完成状态，便于用户确认"今天学到哪"。
- ②区为核心默写区：左为中文释义（含词性），右/下为下划线输入框；题量 50 时须可滚动且输入流畅。
- ③区「校对」按钮常驻底部；批改后在同一区域向下展开报告，错误单词以"释义—用户答案—正确答案"三栏对照，便于即时订正。

## 5. 待确认问题（需客户 / 主理人拍板）

1. **初中词表来源与范围**：客户提供官方初中课标词表，还是由我方内置通用版？是否严格仅"初中"（不含高中）？词表以何种格式（JSON/plist/txt）作为可替换数据文件？后续是否需要更新机制？
2. **词库抽取范围与课程边界**：全书 list 总数为多少（源 PDF 为扫描件，需先 OCR 才能统计，见 Q7）？起点固定 list17 后，学到最后一课如何处理——循环回到前部 / 停止并提示已学完 / 跨轮混合？课程锚定方式：list17 对应**固定发布首日**，还是**用户首次打开日**为首日？
3. **拼写批改容错规则**：校对是否大小写不敏感？是否容忍前后空格、常见拼写变体（英美 colour/color、单复数 s/es）、以及近似拼写错误（如 Levenshtein 距离容错）？是否允许"一个释义多个可接受答案"？
4. **每日词量是否可配置**：50 词为固定值，还是支持 25/50/100 等档位由用户选择？若当日可用非初中词不足 50，取全部并提示，还是放宽到其他 list 补足？
5. **当日课程是否可重做**：当日课程完成后是否锁定（仅一次），还是允许无限次重做以巩固？重做是否覆盖或追加历史报告？
6. **发音需求（P2）是否升级**：客户是否需要发音功能？若需要，是否应提升至 P1？采用系统 TTS 还是内置真人音频资源？
7. **【数据就绪风险】源文件为扫描件**：已核验《雅思词汇绿宝书(1).pdf》为纯图像 PDF（518 页、零可提取文本）。词库抽取**必须走 OCR**，需确认：① 由谁负责 OCR 与人工校对（准确率风险）；② 客户能否直接提供可复制文本版或已有结构化词库以规避该风险；③ OCR 工期与质量是否影响整体排期。
